# SST Desktop — WIKI

Эта вики описывает элементы интерфейса в формате:
**что это**, **для чего нужно**, **как работает**, **на что влияет**, **когда использовать**, **типовые ошибки**.

---

## 0. Версия и обновления

### Элемент: обновление `Stream Subtitle Translator.exe`
- **Что делает:** заменяет исполняемый файл на новую версию.
- **Для чего:** получить исправления и новые функции без ручной пересборки среды.
- **Как работает:** launcher при старте проверяет локальную среду (`.python`, `.venv`, `app-runtime`) и при необходимости доустанавливает отсутствующее.
- **На что влияет:** на доступность новых режимов ASR/перевода/overlay и совместимость с сохранёнными настройками.
- **Пример:** после обновления появился новый параметр перевода — старый конфиг сохраняется, но поле будет работать только в новой версии.

### Элемент: восстановление среды / полный сброс среды
- **Что делает:** чинит или пересоздаёт локальную runtime-среду.
- **Для чего:** если после обновления приложение не запускается или падает при старте.
- **Как работает:** восстановление пытается исправить повреждённые компоненты; полный сброс удаляет локальную среду и устанавливает заново.
- **На что влияет:** может увеличить время первого запуска после операции, потому что компоненты скачиваются/устанавливаются повторно.
- **Когда применять:** сначала восстановление, и только потом полный сброс.

## 1. Быстрый старт

### Элемент: профиль запуска
- **Что делает:** выбирает сценарий старта (Web Speech, NVIDIA, CPU и т.д.).
- **Для чего:** сразу поднять нужный стек без ручной настройки десятков параметров.
- **Как влияет:** задаёт стартовые ограничения и поведение режима распознавания.

### Элемент: `Способ распознавания`
- **Что делает:** выбирает источник ASR (`local`, `browser_google`, `browser_google_experimental`).
- **Для чего:** баланс между качеством, задержкой и требованиями к железу.
- **Пример выбора:**
  - стрим на ноутбуке без GPU -> браузерный режим;
  - стационар с NVIDIA -> локальный Parakeet.

### Элемент: `Язык распознавания` + `Микрофон`
- **Что делает:** задаёт язык декодирования и входной аудио-девайс.
- **На что влияет:** если язык/микрофон выбраны неверно, качество распознавания падает сильнее, чем от большинства тонких настроек.

## 2. Диагностика: если «не работает»

### Сценарий: нет вообще никакого текста
- Проверьте: runtime запущен, выбран корректный микрофон, выдан доступ к микрофону (особенно для браузерного worker-окна).
- Проверьте индикаторы во вкладке диагностики: приходят ли аудиофреймы.

### Сценарий: исходный текст есть, перевода нет
- Проверьте чекбокс `Переводить распознанную речь`.
- Убедитесь, что хотя бы одна линия перевода включена.
- Откройте блок `Переведённый результат`: там отображаются ошибки ключа, endpoint, модели, квот и сети.

### Сценарий: в OBS пусто
- Проверьте URL overlay и что открыт именно `/overlay`.
- Проверьте, что включены нужные видимости (`Показывать оригинальный текст`, `Показывать переводы`).
- Убедитесь, что не выставлены слишком агрессивные TTL (текст может быстро исчезать).

## 3. Профили запуска

### Элемент: `Быстрый запуск через браузерное распознавание`
- **Для чего:** самый простой запуск без локальной AI-модели.
- **Как работает:** создаёт отдельное browser worker-окно, где микрофон выбирается средствами браузера.
- **Ограничение:** после такого старта локальный AI-путь может быть временно залочен до следующего запуска через NVIDIA/CPU профиль.

### Элемент: `Запуск с NVIDIA`
- **Для чего:** максимальная скорость/качество локального Parakeet при доступной CUDA.
- **Как влияет:** выше нагрузка на GPU, ниже задержка до финала.

### Элемент: `Запуск только на CPU`
- **Для чего:** рабочий fallback, когда GPU недоступна.
- **Как влияет:** распознавание заметно медленнее, больше задержка и выше шанс «дрожащих» partial-обновлений.

### Элемент: `Удалённый контроллер` и `Удалённый воркер`
- **Для чего:** split-сценарий controller/worker в LAN.
- **Порядок запуска:** сначала worker, затем controller.
- **Важно:** worker не должен запускаться в browser ASR режимах.

## 4. Распознавание

### Элемент: `Способ распознавания`
- **Что делает:** определяет архитектуру пути «аудио -> текст».
- **Влияние:**
  - `local` — локальный runtime, без зависимости от Web Speech API;
  - `browser_google` — распознавание в браузерном worker;
  - `browser_google_experimental` — экспериментальный браузерный путь с fallback-логикой.

### Элемент: `Язык распознавания`
- **Что делает:** задаёт языковую модель/локаль ASR.
- **Практика:** если стрим двуязычный, стабильнее выбрать язык доминирующей речи и переводить уже в целевые языки.

### Элемент: `Браузер для worker-окна` (desktop)
- **Что делает:** выбирает браузерный движок worker-окна.
- **Для чего:** совместимость с Web Speech API и поведением микрофонных разрешений.
- **Пример:** при проблемах в Chrome можно переключиться на Edge-путь и проверить стабильность сессии.

### Элемент: `Локальный AI-провайдер`
- **Что делает:** выбирает вариант локального Parakeet (`official_eu_parakeet_low_latency`/`official_eu_parakeet`).
- **Влияние:** low-latency профиль быстрее отдаёт результат, но может быть чуть «резче» по стабилизации partial.

## 5. Перевод

### 5.1 Основные чекбоксы

#### Элемент: `Переводить распознанную речь`
- **Что делает:** включает/выключает переводный pipeline.
- **Ключевой эффект:** распознавание работает и без этого чекбокса; выключение перевода не ломает ASR.

#### Элемент: `Использовать кэш переводов`
- **Что делает:** не отправляет в провайдера повторяющиеся фразы.
- **Для чего:** снизить стоимость API и задержку на повторах.
- **Пример:** стример часто повторяет «поехали» — второй и последующие разы перевод может браться мгновенно из кэша.

#### Элемент: `Сохранять кэш переводов на диск`
- **Что делает:** сохраняет кэш между сессиями.
- **Риск:** если часто меняете стиль перевода/промпт модели, старые кэш-ответы могут быть уже не в желаемом стиле.

### 5.2 Линии перевода (слоты)

#### Элемент: `Добавить линию перевода`
- **Что делает:** создаёт целевую языковую линию (до 5).
- **Как влияет:** каждая активная линия — отдельная нагрузка на dispatch и провайдера.

#### Элемент: `Включение/выключение линии`
- **Что делает:** определяет, участвует ли слот в текущей сессии.
- **Практика:** вместо удаления удобно временно выключать линии для быстрого A/B.

#### Элемент: `Провайдер линии`
- **Что делает:** задаёт провайдер конкретно для этого слота.
- **Сценарий:** перевод 1 на `google_translate_v2` (скорость), перевод 2 на LLM-провайдер (стилизация).

### 5.3 Настройки провайдера

#### Элемент: `Провайдер по умолчанию для новых линий`
- **Что делает:** предзаполняет провайдер для новых слотов.
- **Для чего:** не выбирать вручную каждый раз при создании линий.

#### Элемент: поля ключей/адресов/модели
- **Что делает:** задаёт параметры авторизации и маршрутизации запросов.
- **Критично:** ошибка в этих полях проявится в `Переведённый результат` как runtime-ошибка провайдера.

#### Элемент: `Пользовательская инструкция для модели`
- **Что делает:** задаёт стиль/правила перевода для LLM-провайдеров.
- **Влияние:** напрямую меняет тон и формат результата (например, «коротко для субтитров, без пояснений»).

### 5.4 Блок `Переведённый результат`
- **Что делает:** показывает последние переводы и ошибки pipeline.
- **Для чего:** быстрый operational debugging без чтения логов.
- **Что важно:** если перевод опаздывает, это не всегда ошибка — может работать защита от устаревших/суперседированных результатов.

## 6. Вывод субтитров

### 6.1 Элемент: `Схема расположения строк в overlay`
- **`Одна строка`:** максимум компактности, но сложнее читать при 2+ языках.
- **`Две строки`:** базовый компромисс для стримов с оригиналом + одним переводом.
- **`Стек`:** лучшая читаемость многоязычного вывода, но больше вертикального места.

### 6.2 Элемент: `Компактные отступы`
- **Что делает:** уменьшает вертикальные и межстрочные интервалы.
- **Когда использовать:** маленькая область в OBS или плотная верстка сцены.

### 6.3 Элементы видимости
- **`Показывать оригинальный текст`:** включает/выключает исходный язык в overlay.
- **`Показывать переводы`:** общий тумблер для translation-строк.
- **`Максимум строк перевода`:** ограничивает число одновременно видимых переводов по текущему порядку.

### 6.4 Элементы TTL и замены блока
- **`Сколько держать завершённый исходный текст`:** время жизни финального source.
- **`Сколько держать завершённый перевод`:** время жизни финального translation.
- **`Держать исходный, пока виден перевод`:** синхронизирует исчезновение source с still-visible переводом.
- **`Сразу заменять при следующем финале`:** агрессивная смена блока после нового финала.

**Ключевой смысл:** завершённый перевод должен оставаться видимым, пока новая фраза ещё только partial. Замена происходит, когда новая фраза сама стала final и отправлена в перевод.

### 6.5 Элемент: порядок строк (`Выше`/`Ниже`)
- **Что делает:** меняет порядок отрисовки языков/слотов.
- **На что влияет одновременно:** dashboard preview, OBS overlay и режим «первая видимая строка» для OBS Closed Captions.

## 7. Стиль субтитров

### Элемент: выбор пресета
- **Что делает:** применяет набор параметров внешнего вида.
- **Для чего:** быстро переключать оформление под сцену, игру, фон.

### Элемент: параметры стиля (шрифт, размер, цвет, обводка, тень, фон)
- **Как работает:** стиль применяется и к preview, и к overlay (единый источник правды).
- **Практика:** сначала настройте читаемость (контраст/outline), потом эстетику.

### Элемент: слот-ориентированная настройка (`source`, `translation_1...translation_5`)
- **Что делает:** позволяет разный стиль для источника и каждого перевода.
- **Пример:** source — белый с тонкой обводкой; translation_1 — жёлтый; translation_2 — голубой.

### Элемент: сохранение пресета и сохранение config/профиля
- **Важно:** если не сохранить config/профиль, при новом запуске изменения могут не восстановиться.

## 8. OBS Closed Captions

### 8.1 Элемент: `Отправлять субтитры в OBS Closed Captions`
- **Что делает:** включает поток субтитров в OBS WebSocket.
- **Для чего:** нативные CC для платформ/плееров, где это поддерживается.

### 8.2 Элементы подключения (`хост`, `порт`, `пароль`)
- **Как работают:** должны совпадать с настройками OBS WebSocket.
- **Типовая ошибка:** правильный host/port, но неверный пароль -> подключение не установится.

### 8.3 Элемент: `Что отправлять`
- Варианты: live source, final source, translation N, первая видимая строка.
- **Выбор зависит от сценария:**
  - минимальная задержка -> live source;
  - более чистый текст без «дрожания» -> final source;
  - мультиязычный стрим -> translation_1 или первая видимая строка.

### 8.4 Элементы антидребезга и очистки
- **Минимальный интервал partial-обновлений:** ограничивает частоту отправки.
- **Минимальное изменение текста:** отсекает микроскопические правки.
- **Задержка перед заменой финалом:** снижает визуальные рывки.
- **Очистка текста через N мс:** управляет поведением «хвоста» в OBS.
- **Не отправлять дубликаты:** уменьшает шум и лишний трафик.

### 8.5 Элемент: зеркало в текстовый источник OBS
- **Для чего:** отладка и визуальная проверка вне CC-потока.
- **Когда полезно:** если CC не отображается в плеере, но нужно убедиться, что данные реально отправляются.

## 9. Характер распознавания

### Элемент: «быстрота появления текста»
- **Что делает:** сдвигает баланс «раньше показать vs точнее стабилизировать».
- **Влияние:** чем раньше показывать, тем выше вероятность частых правок partial.

### Элемент: «как быстро считать речь завершённой»
- **Что делает:** задаёт агрессивность финализации фразы.
- **Влияние:** слишком быстро -> дробление фраз; слишком медленно -> запаздывающий final.

### Элемент: RNNoise (экспериментально) и сила RNNoise
- **Что делает:** дополнительное шумоподавление перед распознаванием.
- **Когда включать:** только при реально шумном окружении; на чистом сигнале может съедать полезные детали речи.

### Элемент: пресет задержки Parakeet
- **Для чего:** быстро выбрать профиль задержки без ручной правки всех advanced-полей.

## 10. Расширенные настройки распознавания

Этот блок нужен для тонкой калибровки под конкретный микрофон, темп речи и шум.

### Группа: пороги и чувствительность
- Параметры уровня входа/шума и доли речевых кадров решают, что считать «речью», а что «фоном».
- **Симптом завышенных порогов:** пропадает тихая речь.
- **Симптом заниженных порогов:** ложные срабатывания на шум.

### Группа: partial/final поведение
- Параметры частоты обновления, минимального изменения и склейки обновлений управляют стабильностью текста на экране.
- **Практика:** для стрима обычно лучше немного реже, но стабильнее.

### Группа: сегментация фраз
- Настройки пауз и максимальной длины сегмента задают границы финальных фраз.
- От этого напрямую зависит, как перевод распределяется по блокам и как читается в overlay.

### Группа: продвинутое аудио-окно
- Размер окна и overlap управляют контекстом декодирования.
- Неправильный баланс может дать или лишнюю задержку, или потерю устойчивости на границах слов.

**Рекомендация:** менять по одному параметру, фиксировать результат в preview, затем сохранять профиль.

## 11. Замена слов (до перевода)

### Элемент: `Включить замену слов`
- **Что делает:** применяет словарь замен к распознанному тексту до отправки в перевод и до вывода.
- **Для чего:** цензура, исправление имен, бренд-терминов и частых ошибок ASR.

### Элемент: встроенный список мата
- **Что делает:** быстро включает преднастройку фильтра (RU+EN).
- **Влияние:** замены применяются до translation pipeline, поэтому и перевод увидит уже «очищенный» текст.

### Элементы правил (`без учёта регистра`, `только целые слова`)
- **Без учёта регистра:** ловит `Word`, `word`, `WORD`.
- **Только целые слова:** защищает от случайных замен внутри других слов.

### Элементы словаря (`слово/фраза`, `заменить на`, `добавить`, `удалить выбранные`)
- **Пример:** `discord` -> `Дискорд`, чтобы стабилизировать бренд-написание в исходнике и переводах.

## 12. Инструменты и данные

### Элемент: `Глубокая диагностика -> Диагностика runtime`
- **Что показывает:** задержки, состояние ASR, состояние перевода, очередь, browser worker, OBS CC, путь к логам.
- **Для чего:** понять, где именно проблема — аудио, распознавание, перевод, публикация в overlay или вывод в OBS.

### Элемент: блоки config/профилей/экспорта диагностики
- **Config:** текущие активные настройки.
- **Профили:** сохранённые наборы параметров под сценарии.
- **Экспорт диагностики:** пакет для повторяемого разбора проблем.

## 13. Local Parakeet

### Элемент: `Local Parakeet` / `Official EU Parakeet Low Latency`
- **Что это:** локальный AI-режим распознавания.
- **Для чего:** автономный ASR без зависимости от браузерного Web Speech.
- **Когда выбирать:** нужна предсказуемость и контроль в локальной среде, особенно с NVIDIA.

Модельная справка:
- [nvidia/parakeet-tdt-0.6b-v2](https://huggingface.co/nvidia/parakeet-tdt-0.6b-v2)

## 14. Web Speech

### Элемент: worker-страница `/google-asr` (и edge-вариант)
- **Что делает:** запускает распознавание в отдельном окне браузера и отправляет события в основной runtime.
- **Ключевые блоки UI:** старт/стоп, поведение partial/final, диагностика websocket/счётчиков, поля live/final текста.
- **Практический смысл:** если на dashboard пусто, worker-страница помогает понять, приходит ли распознавание до backend.

---

## 15. Web Speech (Experimental)

### Элемент: worker-страница `/google-asr-experimental` (и edge-вариант)
- **Что делает:** использует экспериментальный путь запуска Web Speech.
- **Зачем нужен:** тестировать сценарии, где классический путь нестабилен.
- **Особенность:** при несовместимости возможен fallback к обычному запуску.

## 17. Глоссарий

- **Промежуточный текст (partial):** черновой распознанный текст, который ещё может меняться.
- **Финальный текст (final):** зафиксированный сегмент, который считается завершённой фразой.
- **Слот перевода:** отдельная целевая линия `translation_1..translation_5` со своими параметрами.
- **Overlay:** веб-страница `/overlay` для OBS Browser Source.
- **OBS Closed Captions:** поток скрытых субтитров, отправляемый в OBS WebSocket.
