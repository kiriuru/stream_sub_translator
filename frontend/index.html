      Recognition Notes
      
            
      Lower partial emit and silence hold values feel faster, but can increase flicker.
      
            
      `Pause to finalize` controls when a spoken phrase becomes one completed subtitle block.
      
            
      `Chunk window` and `chunk overlap` remain advanced compatibility controls. Leave them at `0` unless you are testing a specific backend tuning case.
      
            
      `Min RMS` and `Min voiced ratio` are optional filters for quiet room noise and weak false starts. If recognition quality drops, turn the quiet-input gate back off.
      
            
      VAD modes: 0 = fastest and most sensitive, 1 = responsive balanced, 2 = stricter and calmer, 3 = strictest and least likely to trigger on noise.

      Overview, status, and first launch flow
      
                
      This topic explains the always-visible top part of the dashboard. It is the control center you use before opening deeper feature tabs.
      
                
      
                  
      Health badge
      Shows whether the local FastAPI app is responding. If health is not OK, other controls can be stale.
      
                  
      Runtime badge
      Shows the lifecycle of the recognition runtime: idle, starting, listening, transcribing, translating, or error.
      
                  
      ASR provider / device / mode badges
      Summarize the active recognition provider, selected microphone, partial-text behavior, and whether the app is in local or Web Speech mode.
      
                  
      Start
      Loads the selected runtime and begins listening. Local AI modes may need model/runtime bootstrap before they start.
      
                  
      Stop
      Stops the current runtime. Use Stop/Start after changing microphone, ASR provider, or runtime profile choices.
      
                  
      Recognition method
      Chooses Local Parakeet, Web Speech, or Web Speech (Experimental). Worker mode disables browser recognition by design.
      
                  
      Microphone
      Selects the local input device for local capture. Web Speech microphone switching happens in the browser worker address-bar permission UI.
      
                  
      Subtitle Output Preview
      Shows the same ordering and visibility rules that the OBS overlay receives.
      
                  
      Global Save
      Saves most dashboard settings to local config. Some settings apply instantly; runtime-affecting settings need Stop/Start.

      Speech input
      
                
      Recognition modes and the Tuning tab
      
                
      This section describes the recognition method selector and the user-facing Tuning controls. Exact millisecond-level ASR parameters live in Tools & Data.
      
                
      
                  
      Local Parakeet
      Main local AI path. It uses local microphone capture, VAD, and the selected Parakeet provider. GPU is preferred when available.
      
                  
      Web Speech
      Opens a visible browser worker window. The browser performs recognition and sends partial/final text to the app over WebSocket.
      
                  
      Web Speech (Experimental)
      Uses a newer audio-track startup path. Keep it for compatibility experiments, not as the most stable default.
      
                  
      Recognition language
      Language hint for browser recognition modes. Local providers may use their own model/language behavior.
      
                  
      How quickly text appears
      Tuning slider for the first live text. Higher values make partial text appear sooner; lower values wait for more speech before showing text.
      
                  
      How quickly speech is considered finished
      Tuning slider for phrase finalization. Higher values close phrases faster after a pause; lower values keep the phrase open longer.
      
                  
      How stable or less chatty updates should be
      Tuning slider for partial rewrite calmness. Higher values reduce tiny text changes and flicker; lower values feel more live but can rewrite more often.
      
                  
      RNNoise noise reduction
      Optional experimental noise reduction. Leave it off unless background noise causes false starts or dirty recognition.
      
                  
      RNNoise strength
      Blend amount for the RNNoise output. 0% keeps the raw path; higher values apply stronger noise-reduced audio and can change recognition behavior.
      
                  
      Tuning notes
      The Tuning tab works on top of local microphone capture, VAD, and websocket updates. Save config or profile after finding stable values.
      
                
      
              
      
              
      
                
      Вход речи
      
                
      Режимы распознавания и вкладка Tuning
      
                
      Этот раздел описывает выбор способа распознавания и пользовательские настройки вкладки Tuning. Точные ASR-параметры в миллисекундах находятся в Tools & Data.
      
                
      
                  
      Local Parakeet
      Основной локальный AI-путь. Использует локальный захват микрофона, VAD и выбранный Parakeet provider. При наличии CUDA целевой путь — GPU.
      
                  
      Web Speech
      Открывает видимое окно browser worker-а. Браузер сам распознаёт речь и отправляет partial/final текст в приложение по WebSocket.
      
                  
      Web Speech (Experimental)
      Пробует новый запуск через audio track. Это режим для проверки совместимости, а не самый стабильный default.
      
                  
      Recognition language
      Подсказка языка для browser recognition. Локальные провайдеры могут использовать свои правила модели/языка.
      
                  
      Как быстро появляется текст
      Ползунок Tuning для первого live-текста. Больше — partial появляется раньше; меньше — приложение ждёт больше речи перед показом текста.
      
                  
      Как быстро речь считается законченной
      Ползунок Tuning для завершения фразы. Больше — фраза быстрее закрывается после паузы; меньше — фраза дольше остаётся открытой.
      
                  
      Насколько спокойно обновляется текст
      Ползунок Tuning для стабильности partial-текста. Больше — меньше мелких переписываний и мерцания; меньше — ощущается живее, но чаще меняет текст.
      
                  
      RNNoise шумоподавление
      Опциональное экспериментальное шумоподавление. Оставляйте выключенным, пока фон не вызывает ложные старты или грязное распознавание.
      
                  
      Сила RNNoise
      Степень подмешивания результата RNNoise. 0% оставляет сырой путь; большие значения сильнее меняют входной звук и могут менять поведение распознавания.
      
                  
      Пояснения по Tuning
      Вкладка Tuning работает поверх локального захвата микрофона, VAD и websocket-обновлений. После подбора стабильных значений сохраните config или профиль.
      
                
      
              
      
            
      

            
      
              
      
                
      Language output
      
                
      Translation routing and provider settings
      
                
      Translation is optional. The app can show source only, translations only, source plus one translation, or source plus several translations.
      
                
      
                  
      Translate recognized speech
      Master switch for sending finalized source text into the translation queue.
      
                  
      Translation line
      A visible/output slot. Each line has its own enabled state, target language, provider, and label.
      
                  
      Target language
      The language requested from the provider for that line. Duplicate target languages are allowed for different providers or labels.
      
                  
      Provider for this line
      Allows mixed routing, for example one line through Google and another through a local LLM provider.
      
                  
      Line label
      Short label used to identify the line in UI/output contexts.
      
                  
      Default provider
      Provider preselected for newly added translation lines.
      
                  
      Provider settings selector
      Chooses which provider's shared API key, URL, model, or prompt settings are being edited.
      
                  
      API key
      Secret token for providers that require one. It stays local in config/profiles.
      
                  
      Base URL / Provider URL / Endpoint
      Provider-specific server address. For local LLM providers this usually points to a local OpenAI-compatible endpoint.
      
                  
      Region
      Cloud-specific routing field, used by providers such as Azure Translator.
      
                  
      Model
      LLM model name for OpenAI-compatible providers such as OpenAI, OpenRouter, LM Studio, and Ollama.
      
                  
      Custom prompt override
      Optional LLM prompt replacement. Leave empty unless you need custom subtitle translation behavior.
      
                  
      Move Up / Move Down
      Changes output order. The same order is used by preview, overlay, and first-visible-line OBS caption mode.
      
                
      
              
      
              
      
                
      Языковой вывод
      
                
      Маршрутизация перевода и настройки провайдеров
      
                
      Перевод опционален. Приложение может показывать только оригинал, только переводы, оригинал плюс один перевод или оригинал плюс несколько переводов.
      
                
      
                  
      Translate recognized speech
      Главный переключатель: отправлять ли завершённый source-текст в очередь перевода.
      
                  
      Translation line
      Слот вывода. У каждой линии есть своё включение, целевой язык, провайдер и метка.
      
                  
      Target language
      Язык, который запрашивается у провайдера для этой линии. Дубли языков допустимы, например для разных провайдеров.
      
                  
      Provider for this line
      Позволяет смешивать маршруты: например, одна линия через Google, другая через локальный LLM.
      
                  
      Line label
      Короткая метка линии для UI/output-контекстов.
      
                  
      Default provider
      Провайдер, который будет выбран по умолчанию для новых линий перевода.
      
                  
      Provider settings selector
      Выбирает, настройки какого провайдера сейчас редактируются: API key, URL, модель или prompt.
      
                  
      API key
      Секретный токен для провайдеров, которым он нужен. Остаётся локально в config/profiles.
      
                  
      Base URL / Provider URL / Endpoint
      Адрес сервера провайдера. Для локальных LLM обычно указывает на локальный OpenAI-compatible endpoint.
      
                  
      Region
      Поле облачной маршрутизации, например для Azure Translator.
      
                  
      Model
      Имя LLM-модели для OpenAI-compatible провайдеров: OpenAI, OpenRouter, LM Studio, Ollama.
      
                  
      Custom prompt override
      Необязательная замена LLM prompt-а. Оставляйте пустым, если не нужен специальный стиль перевода субтитров.
      
                  
      Move Up / Move Down
      Меняет порядок вывода. Этот же порядок используют preview, overlay и first-visible-line режим OBS captions.
      
                
      
              
      
            
      

            
      
              
      
                
      Presentation
      
                
      Subtitle routing, lifetime, and visual style
      
                
      
                  
      Overlay layout preset
      Chooses the base overlay arrangement: single, dual-line, stacked, or compact-compatible presentation.
      
                  
      Compact overlay spacing
      Reduces overlay gaps and padding for denser layouts.
      
                  
      Show source / show translations
      Controls whether original speech and translated lines are visible independently.
      
                  
      Maximum translated lines
      Caps how many translation slots can appear at once. The renderer supports up to five translated lines.
      
                  
      Source hold time
      How long completed original text remains visible after finalization.
      
                  
      Translation hold time
      How long completed translations remain visible. This can outlive the source text.
      
                  
      Sync expiry
      Keeps source visible while its translation is still visible, useful for paired source+translation reading.
      
                  
      Allow early replace
      Allows the next finalized phrase to replace the previous completed block before TTL naturally expires.
      
                  
      Preset
      Starting visual style. Built-in presets are editable; custom presets are saved locally.
      
                  
      Base style
      Default font, size, color, outline, shadow, background, alignment, spacing, and animation effect for all lines.
      
                  
      Slot overrides
      Per-line overrides for source and translation_1 through translation_5. Use them when each language needs different emphasis.
      
                  
      Project-local fonts
      Fonts bundled with the project are exposed through the app and are more reliable than browser-dependent system font enumeration.
      
                
      
              
      
              
      
                
      Подача на экран
      
                
      Маршрутизация, время жизни и стиль субтитров
      
                
      
                  
      Overlay layout preset
      Выбирает базовую раскладку overlay: single, dual-line, stacked или совместимую compact-подачу.
      
                  
      Compact overlay spacing
      Уменьшает отступы overlay для более плотной раскладки.
      
                  
      Show source / show translations
      Независимо управляет видимостью оригинальной речи и переводов.
      
                  
      Maximum translated lines
      Ограничивает число строк перевода на экране. Renderer поддерживает до пяти переводных строк.
      
                  
      Source hold time
      Сколько завершённый оригинальный текст остаётся видимым после final.
      
                  
      Translation hold time
      Сколько завершённые переводы остаются видимыми. Перевод может жить дольше source-текста.
      
                  
      Sync expiry
      Держит source видимым, пока виден его перевод. Полезно для чтения пары оригинал+перевод.
      
                  
      Allow early replace
      Разрешает следующей final-фразе заменить предыдущий готовый блок до естественного истечения TTL.
      
                  
      Preset
      Стартовый визуальный стиль. Встроенные пресеты можно редактировать, custom presets сохраняются локально.
      
                  
      Base style
      Общие шрифт, размер, цвет, обводка, тень, фон, выравнивание, интервалы и эффект анимации для всех строк.
      
                  
      Slot overrides

      Maintenance
      
                
      Tools, data, profiles, logs, and diagnostics
      
                
      
                  
      Latency metrics
      Shows timing information for the live pipeline. Use it to spot whether delay comes from recognition, translation, websocket delivery, or output rendering.
      
                  
      ASR diagnostics
      Reports recognition-side state: selected mode/provider, runtime status, device/runtime details, and backend messages when available.
      
                  
      Translation diagnostics
      Shows translation queue/provider state. If translation appears late or missing, check whether the queue is idle, busy, disabled, or failing.
      
                  
      Translation queue idle
      Means no translation job is currently waiting or running. If you expected output, check whether translation is enabled and lines are configured.
      
                  
      Browser worker diagnostics
      Shows whether the Web Speech worker window is connected. If it says not connected, browser modes cannot feed text into the app.
      
                  
      OBS Closed Captions diagnostics
      Shows OBS caption connection/output state separately from the visible overlay.
      
                  
      Logs location
      Reminder of where local logs are written: backend log, runtime events, and latest session JSONL.
      
                  
      VAD mode
      Speech detector sensitivity. 0 is fastest and easiest to trigger; 3 is strictest and rejects more noise.
      
                  
      Partial emit interval
      Minimum time between live text updates. Lower feels faster but can flicker more.
      
                  
      Min speech
      How much speech is needed before recognition starts producing text for the current segment.
      
                  
      Silence hold
      How long a short pause keeps the current phrase open instead of splitting it.
      
                  
      Pause to finalize
      How long a pause must last before the current phrase becomes final and enters translation/subtitle routing.
      
                  
      Hard max phrase
      Maximum phrase length before forced finalization/splitting prevents endless segments.
      
                  
      Minimum text change
      Small partial rewrites below this character delta are suppressed to reduce flicker.
      
                  
      Partial coalescing
      Groups rapid tiny partial updates so the UI and overlay receive calmer text.
      
                  
      Chunk window / overlap
      Advanced compatibility controls. Keep 0/0 unless testing a specific backend behavior.
      
                  
      Ignore very quiet input
      Optional quiet-input gate before ASR. Use only if room noise causes false starts.
      
                  
      Min RMS
      Energy threshold for the quiet-input gate. Too high can hide real speech.
      
                  
      Min voiced ratio
      Minimum share of frames that must look speech-like before ASR receives the segment.
      
                  
      First partial speech
      How much speech must be collected before the first live partial text can be emitted.
      
                  
      Live Events
      Client-side runtime feed. Useful for seeing what the dashboard is receiving without opening backend logs.
      
                  
      Local Config JSON
      Current merged dashboard config. Use it for inspection or manual import/export workflows.
      
                  
      Save Current Config
      Writes the current settings into local user data.
      
                  
      Export Config
      Downloads the current config as JSON so it can be archived or moved.
      
                  
      Import Config
      Loads a JSON config file into the dashboard. Review settings before starting runtime.
      
                  
      Export Diagnostics
      Bundles local logs and state useful for troubleshooting without exposing cloud services.
      
                  
      Profiles
      Named saved configurations. Use profiles for different stream setups, language combinations, or OBS scenes.
      
                  
      Logs location
      Backend and runtime event logs live under local project/user-data paths and stay on this machine.

      Latency metrics

      VAD mode

      Partial emit interval
      Минимальный интервал между live-обновлениями текста. Меньше — быстрее, но больше риск мерцания.
      
                  
      Min speech
      Сколько речи нужно до того, как распознавание начнёт выдавать текст для текущего сегмента.
      
                  
      Silence hold
      Сколько короткая пауза держит текущую фразу открытой, не разделяя её.
      
                  
      Pause to finalize
      Сколько должна длиться пауза, чтобы фраза стала final и ушла в перевод/маршрутизацию субтитров.
      
                  
      Hard max phrase
      Максимальная длина фразы до принудительного завершения, чтобы сегмент не рос бесконечно.
      
                  
      Minimum text change
      Мелкие переписывания partial-текста меньше этого порога подавляются, чтобы уменьшить мерцание.
      
                  
      Partial coalescing
      Склеивает частые мелкие partial-обновления, чтобы UI и overlay получали более спокойный текст.
      
                  
      Chunk window / overlap
      Продвинутые параметры совместимости. Оставляйте 0/0, если не тестируете конкретное поведение backend-а.
      
                  
      Ignore very quiet input
      Опциональный quiet-input gate перед ASR. Используйте только если комнатный шум вызывает ложные старты.
      
                  
      Min RMS
      Порог громкости для quiet-input gate. Слишком высокое значение может скрыть настоящую речь.
      
                  
      Min voiced ratio
      Минимальная доля кадров, похожих на речь, перед тем как сегмент попадёт в ASR.
      
                  
      First partial speech
      Сколько речи нужно накопить до отправки первого live partial-текста.
      
                  
      Live Events
      Клиентский поток runtime-событий. Помогает увидеть, что получает dashboard, не открывая backend logs.
      
                  
      Local Config JSON

      Save Current Config
      Записывает текущие настройки в локальные user data.

      LAN mode
      
                
      Desktop profiles and Remote Controller / Worker parameters
      
                
      Default startup is local-first and localhost-only. Remote mode is an explicit LAN controller/worker split for the same local product, not an internet service.
      
                
      Reliable startup order
      
                
        
                  
      1. Start the Worker machine first with the Remote Worker desktop profile or `start-remote-worker.bat`. Confirm it is reachable on the LAN and note its IP/port.
        2. 
                  
      2. Start the Controller machine with the Remote Controller profile or `start-remote-controller.bat`. This is the machine where the microphone and main dashboard are controlled.
        3. 
                  
      3. In Tools & Data, enter the Worker Base URL on the controller, for example `http://192.168.1.50:8765`.
        4. 
                  
      4. Click Check Worker Health. Do not continue until health succeeds; otherwise fix firewall, IP, port, or worker startup first.
        5. 
                  
      5. Click Create Local Pair on the controller, then use the generated Session ID / Pair Code with the worker-side pairing flow if prompted.
        6. 
                  
      6. Click Refresh Remote State and verify the role/session/online fields look correct.
        7. 
                  
      7. Click Sync Worker Settings so the worker receives controller settings and is forced into local AI ASR mode.
        8. 
                  
      8. Click Prepare Remote Run. This aligns pairing/session context before audio starts flowing.
        9. 
                  
      9. Click Start Worker Runtime, then Check Worker Runtime until it reports running/listening or a clear status.
        10. 
                  
      10. Open the Controller Bridge and Worker Bridge windows if the profile did not open them automatically. Keep bridge windows open while using remote mode.
        11. 
                  
      11. Press Start on the main dashboard/controller side to begin microphone capture and remote audio/result flow.
        12.

      Pair Code
      Short verification code used during pairing so the controller and worker agree on the intended session.
      
                  
      Create Local Pair
      Creates/refreshes local pairing state for controller-side setup.
      
                  
      Refresh Remote State
      Reads current remote mode state: enabled flag, role, session, active status, and online flags.
      
                  
      Sync Worker Settings
      Sends relevant controller settings to the worker. Worker sync forces AI/local ASR mode and prevents drift into browser-worker modes.
