Speech input

Recognition modes and the Tuning tab

This section describes the recognition method selector and the user-facing Tuning controls. Exact millisecond-level ASR parameters live in Tools & Data.

Local Parakeet
Main local AI path. It uses local microphone capture, VAD, and the selected Parakeet provider. GPU is preferred when available.
Web Speech
Opens a visible browser worker window. The browser performs recognition and sends partial/final text to the app over WebSocket.
Web Speech (Experimental)
Uses a newer audio-track startup path. Keep it for compatibility experiments, not as the most stable default.
Recognition language
Language hint for browser recognition modes. Local providers may use their own model/language behavior.
How quickly text appears
Tuning slider for the first live text. Higher values make partial text appear sooner; lower values wait for more speech before showing text.
How quickly speech is considered finished
Tuning slider for phrase finalization. Higher values close phrases faster after a pause; lower values keep the phrase open longer.
How stable or less chatty updates should be
Tuning slider for partial rewrite calmness. Higher values reduce tiny text changes and flicker; lower values feel more live but can rewrite more often.
RNNoise noise reduction
Optional experimental noise reduction. Leave it off unless background noise causes false starts or dirty recognition.
RNNoise strength
Blend amount for the RNNoise output. 0% keeps the raw path; higher values apply stronger noise-reduced audio and can change recognition behavior.
Tuning notes
The Tuning tab works on top of local microphone capture, VAD, and websocket updates. Save config or profile after finding stable values.

Вход речи

Режимы распознавания и вкладка Tuning

Этот раздел описывает выбор способа распознавания и пользовательские настройки вкладки Tuning. Точные ASR-параметры в миллисекундах находятся в Tools & Data.

Local Parakeet
Основной локальный AI-путь. Использует локальный захват микрофона, VAD и выбранный Parakeet provider. При наличии CUDA целевой путь — GPU.
Web Speech
Открывает видимое окно browser worker-а. Браузер сам распознаёт речь и отправляет partial/final текст в приложение по WebSocket.
Web Speech (Experimental)
Пробует новый запуск через audio track. Это режим для проверки совместимости, а не самый стабильный default.
Recognition language
Подсказка языка для browser recognition. Локальные провайдеры могут использовать свои правила модели/языка.
Как быстро появляется текст
Ползунок Tuning для первого live-текста. Больше — partial появляется раньше; меньше — приложение ждёт больше речи перед показом текста.
Как быстро речь считается законченной
Ползунок Tuning для завершения фразы. Больше — фраза быстрее закрывается после паузы; меньше — фраза дольше остаётся открытой.
Насколько спокойно обновляется текст
Ползунок Tuning для стабильности partial-текста. Больше — меньше мелких переписываний и мерцания; меньше — ощущается живее, но чаще меняет текст.
RNNoise шумоподавление
Опциональное экспериментальное шумоподавление. Оставляйте выключенным, пока фон не вызывает ложные старты или грязное распознавание.
Сила RNNoise
Степень подмешивания результата RNNoise. 0% оставляет сырой путь; большие значения сильнее меняют входной звук и могут менять поведение распознавания.
Пояснения по Tuning
Вкладка Tuning работает поверх локального захвата микрофона, VAD и websocket-обновлений. После подбора стабильных значений сохраните config или профиль.

LAN mode

Desktop profiles and Remote Controller / Worker parameters

Default startup is local-first and localhost-only. Remote mode is an explicit LAN controller/worker split for the same local product, not an internet service.

Reliable startup order

  1. Start the Worker machine first with the Remote Worker desktop profile or `start-remote-worker.bat`. Confirm it is reachable on the LAN and note its IP/port.
  2. Start the Controller machine with the Remote Controller profile or `start-remote-controller.bat`. This is the machine where the microphone and main dashboard are controlled.
  3. In Tools & Data, enter the Worker Base URL on the controller, for example `http://192.168.1.50:8765`.
  4. Click Check Worker Health. Do not continue until health succeeds; otherwise fix firewall, IP, port, or worker startup first.
  5. Click Create Local Pair on the controller, then use the generated Session ID / Pair Code with the worker-side pairing flow if prompted.
  6. Click Refresh Remote State and verify the role/session/online fields look correct.
  7. Click Sync Worker Settings so the worker receives controller settings and is forced into local AI ASR mode.
  8. Click Prepare Remote Run. This aligns pairing/session context before audio starts flowing.
  9. Click Start Worker Runtime, then Check Worker Runtime until it reports running/listening or a clear status.
  10. Open the Controller Bridge and Worker Bridge windows if the profile did not open them automatically. Keep bridge windows open while using remote mode.
  11. Press Start on the main dashboard/controller side to begin microphone capture and remote audio/result flow.

running is the worker process state; status is the worker runtime lifecycle; message carries the worker-side error/progress text when available.

LAN режим

Desktop profiles и параметры Remote Controller / Worker

Обычный запуск local-first и localhost-only. Remote mode — это явный LAN split controller/worker для того же локального продукта, не интернет-сервис.

Надёжный порядок запуска

  1. Сначала запустите Worker-машину через desktop profile Remote Worker или `start-remote-worker.bat`. Убедитесь, что она доступна в LAN, и запомните IP/порт.
  2. Потом запустите Controller-машину через Remote Controller profile или `start-remote-controller.bat`. Это машина, где находится микрофон и основная панель управления.
  3. В Tools & Data на controller-е введите Worker Base URL, например `http://192.168.1.50:8765`.
  4. Нажмите Check Worker Health. Не продолжайте, пока health не успешен; иначе сначала исправьте firewall, IP, port или запуск worker-а.
  5. Нажмите Create Local Pair на controller-е, затем используйте созданные Session ID / Pair Code в worker-side pairing flow, если он запрошен.
  6. Нажмите Refresh Remote State и проверьте, что role/session/online поля выглядят правильно.
  7. Нажмите Sync Worker Settings, чтобы worker получил настройки controller-а и был принудительно выставлен в local AI ASR mode.
  8. Нажмите Prepare Remote Run. Это выравнивает pairing/session context перед передачей audio.
  9. Нажмите Start Worker Runtime, затем Check Worker Runtime до состояния running/listening или понятного status.
  10. Откройте Controller Bridge и Worker Bridge windows, если профиль не открыл их автоматически. Держите bridge windows открытыми во время remote mode.
  11. Нажмите Start в основной dashboard/controller панели, чтобы начать захват микрофона и remote audio/result flow.
Web Speech quick start
Desktop profile, который может пропустить bootstrap локального AI и использовать browser recognition worker.
NVIDIA GPU
Desktop profile для локального AI-распознавания с CUDA/GPU acceleration.
CPU-only
Локальный AI fallback profile. Ожидаемо медленнее и считается degraded mode.
Remote Controller
Сторона dashboard/control. Захватывает микрофон локально, отправляет звук LAN worker-у и получает обратно transcript/translation events.
Remote Worker
Сторона ASR processing на другой LAN-машине. Должна использовать local AI runtime, а не Web Speech.
Worker Base URL
HTTP-адрес worker app, например http://192.168.1.50:8765. Используйте LAN IP worker-машины и порт, с которым она запущена.
Session ID
Идентификатор remote session, общий для controller и worker. Связывает signaling, audio ingest и result ingest в один run.
Pair Code
Короткий verification code при pairing, чтобы controller и worker подтвердили одну и ту же session.
Create Local Pair
Создаёт или обновляет локальное pairing state на стороне controller-а.
Refresh Remote State
Читает текущее состояние remote mode: enabled, role, session, active и online-флаги.
Sync Worker Settings
Отправляет нужные настройки controller-а на worker. Sync принудительно держит worker в AI/local ASR mode и не даёт уйти в browser-worker пути.
Check Worker Health
Проверяет health endpoint worker-а. Используйте первым, если controller не может достучаться до worker-а.
Check Worker Runtime
Показывает, запущен ли runtime на worker-е и какой status/message он сообщает.
Prepare Remote Run
Делает подготовку перед remote run: pairing/session context и выравнивание worker-side settings.
Start Worker Runtime
Запускает recognition runtime на worker-е. Controller уже должен знать worker URL/session.
Stop Worker Runtime
Останавливает worker-side recognition runtime, не меняя обычный local-first default приложения.
Open Local Worker Bridge
Открывает локальную worker bridge страницу, через которую идёт remote audio/result flow.
Open Controller Bridge
Открывает controller bridge страницу, которая участвует в LAN signaling и event relay.
Remote state fields
enabled — remote mode настроен; role — controller/worker/disabled; active — session сейчас активна; controller_online и worker_online — heartbeat presence.
Worker runtime fields
running — состояние процесса worker-а; status — lifecycle runtime; message — worker-side error/progress текст, если доступен.
